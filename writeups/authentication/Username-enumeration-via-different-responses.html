Mindset & Strategy

When approaching a login page, your first goal is to observe how the server communicates. In this lab, the objective was to determine if the application reveals too much information about its users. By submitting random credentials, we checked if the error messages were specific (e.g., "Invalid password" vs. "Invalid username"). Since the application did not implement rate limiting (account lockout or IP blocking), it allowed us to use Brute Force techniques. By identifying a unique error message for a valid username, we could narrow down our target and eventually find the correct password.

step-by-step-Execution

1. Intercept the Request:

With Burp Suite running, attempt a login with dummy credentials. Capture the POST /login request and send it to Repeater to analyze the server's behavior.

2. Identify the Vulnerability: Notice that the error message changes depending on whether the username exists. We will exploit this using Intruder.

3. Username Enumeration:

Send the request to Intruder and set the payload position on the username parameter.

In the Payloads tab, paste the username list provided in the lab description.

Launch the attack and sort the results by length. A response with a different length indicates a valid username (likely because the error message changed from "Invalid username" to "Invalid password").

4. Password Brute Force:

Take the identified valid username and go back to the Positions tab.

Fix the username parameter to the valid one you found and set the papayload position on the password parameter.

Paste the password list into the Payloads configuration.

5. Final Exploitation:

Start the attack and again sort by Length or Status Code (look for a 302 Found or a different response length).

Once you find the correct credential, right-click the request and select "Show response in browser" or manually log in.
