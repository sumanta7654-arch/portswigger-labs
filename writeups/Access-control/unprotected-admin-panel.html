Access Control | Apprentice

Lab Description

This lab has an unprotected admin panel that is hidden from public view. The goal is to locate this panel and use it to delete the user carlos.

To solve the lab, we will use common reconnaissance techniques to find the secret URL that the developers thought was hidden.

Mindset & Strategy

Developers often use robots.txt to tell search engines (like Google) which pages not to crawl. Ironically, this file often acts as a roadmap for hackers, as it explicitly lists hidden or sensitive directories.

Our strategy is to check these "disallowed" paths. If the server only relies on the URL being hard to guess but doesn't implement actual authentication, we can gain full administrative access just by visiting the link.

Step-By-Step-Execution

1. Reconnaissance: Go to the lab's home page or login screen. Instead of trying to guess credentials, we will look for hidden files. Manually navigate to the /robots.txt file by appending it to the root URL.

2. Analyze robots.txt: You will see a "Disallow" entry. In this lab, it reveals a hidden path: Disallow: /administrator-panel. This is our target.

3. Access the Panel: Copy the path /administrator-panel and paste it into your browser's address bar after the main lab URL. You will be granted access to the administrative dashboard without being asked for a password.

4. Exploit & Solve: Locate the user management section, find the user carlos, and click the Delete button. The lab is now solved.

Technical Explanation

The vulnerability here is Unprotected Admin Functionality combined with Information Disclosure. The application assumes that as long as the URL is not linked on the main page, it is secure. However, using robots.txt to hide directories is a major security risk because it is a publicly readable file that serves as a directory of sensitive areas.

How To Fix

°      Enforce Authentication: Never rely on "hidden" URLs for security. Every administrative page must require a valid, authenticated session with administrative privileges.

°      Role-Based Access Control (RBAC): Implement strict checks on the backend to ensure that even if a user finds the URL, they cannot perform actions without the proper role.

°      Secure robots.txt: Do not list sensitive internal directories in robots.txt. Use proper server-side access controls instead.
